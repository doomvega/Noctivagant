Recently, I've expanded my efforts to move away from social media. This should not surprise anyone; pretty much my entire deal since figuring out webmastery was taking back the web from corporate hands. I've done some really neat things lately, mostly in the realm of making my websites easier to work on. (Some of it was a pain in the ass, but it was worth it.)

Noctivagant's blog operates like this:

  1. I write posts by hand, directly into HTML.
  2. I manually update an RSS feed.

It's incredibly easy, allows for me to write whatever I want, and feels home-made. I love doing it.

Doing this directly into the Neocities editor is fine, but it feels a little janky to me. I should be able to connect to my website's code remotely and edit from any computer available to me, without needing to manually update a GitHub repo every time I do so. Also, I'm really tired of needing my tower to do anything with my website. Most of what I've built for Noctivagant, I've built on my tower. No big deal, but when you're always in pain and always exhausted, it's so much easier to just flip open a laptop and get coding.

Luckily, there are ways to do this.

You can hook up VSCode to GitHub. This system works best if you don't have an existing GitHub repository for your project. (I had to remake a ton of repos for all of the sites I maintain.) If you do have an existing repo, you can connect to it using VSCode's "connect to..." function. Just paste the URL of your repo and it should pull the code for you to tinker with. You can then save this as a workspace so you can open and alter whenever.

Pushing updates from GitHub to Neocities is a little bit more technical (and certainly annoying if you have a ton of sites like I do), but it takes only a few minutes and some testing to make sure your updates are getting pushed correctly. You should be able to add this system to an existing repo, but I've found that it's easier to just make a new repo, just like with the system above.

This means that updating my blog and RSS feed are simultaneous rather than one after the other, like is usual in the Neocities code editor. I've found this system to be smooth, rewarding, and relatively easy to implement, and it makes the process of tending to my sites much simpler. It's actually made me reconsider some of my blogging spots. I love BearBlog, but doing this is cool too. I'm not sure what to do here if the goal is trimming down my social media! And don't even get me started on Zonelets. I've been considering making a Zonelets blog, but then what would I do with this one? Would I just archive and replace it? Zonelets is so fun to work with remotely...

All of this is to say making websites is awesome, and if you can get a system going that makes working on your sites a little easier despite the horrors, you should do it. I'm not even joking when I say that working on my sites like this has made me vastly reconsider my social media usage in its entirety.
